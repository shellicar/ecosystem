## Vite 8 OXC decorator bug

OXC cannot parse the `@dependsOn` decorator combined with definite assignment (`!`) syntax. `SyntaxError: Invalid or unexpected token` on all test files. Vite 7 works. This blocks vite 8 until OXC fixes it.

Dangerous peer resolution trap: vitest resolves vite as a peer dependency. If vite 8 ends up anywhere in the lockfile, it can leak into vitest's peer resolution for packages that don't declare vite directly. `pnpm why vite` is the tool to check. `pnpm add vite@^7` does NOT remove vite 8 from peer resolutions already in the lockfile.

## Factory singleton cache key: `factory ?? implementation`, not descriptor

The singleton cache in core-di (and core-di-lite) is keyed by `factory ?? implementation`, not by the descriptor object. Using the descriptor would break scoped providers: `ServiceCollection.clone()` does `{ ...descriptor }` (shallow clone), producing a new object reference. Scoped providers inherit the parent's singleton map, so cloned descriptors would miss cache entries.

`factory ?? implementation` means: same factory function reference or same class = same singleton. Different factories on the same class = different singletons. This is the intended semantics.

Circular dependency detection stays keyed by identifier, not cacheKey. The `sharedSingletons` check happens before marking anything as resolving.

## Syncpack version group order matters

Version groups are evaluated first-match-wins. Ignores for specific packages must be declared before the catch-all default group, or they'll never be reached.

## `workspace:^` vs npm version in examples

Example packages must use `"workspace:^"` for local `@shellicar/*` packages, not `"^1.2.3"`. With a version specifier, pnpm resolves peer metadata from the published npm package instead of the local workspace package. This caused vite peer deps to resolve incorrectly during the syncpack pass.

## tsconfig.check.json is a symlink

Every package has `tsconfig.check.json` as a symlink to `../../tsconfig.check.json`. The `Find` tool does not follow symlinks, so the file appears missing when scanning a package directory. Use `ls -la` to detect it.

## Tag format transition

The monorepo uses `<package-name>@<version>` tags (e.g., `core-di@4.0.0`). Older `changes.jsonl` entries reference bare version tags (e.g., `1.3.2`) from pre-monorepo history. Those tags don't exist in this repo. The changelog generator handles both formats: old ones link to the tag-only URL, new ones to `package@version`.

## core-di-lite: symbol keys need Reflect.ownKeys

`@dependsOn` stores metadata with `ctx.name` as the key, which is `string | symbol`. `Object.entries` only iterates string keys, silently dropping symbol-keyed fields. `Reflect.ownKeys` is required in `injectDependencies`.
