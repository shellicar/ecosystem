## pnpm overrides land in pnpm-workspace.yaml, not package.json

`pnpm audit --fix` writes overrides to `pnpm-workspace.yaml` in this monorepo. The script output says "added N overrides to package.json" but that's misleading. The file to check is always `pnpm-workspace.yaml`.

## Protobufjs has two separate version ranges in overrides

The existing override `protobufjs@<7.5.5: '>=7.5.5'` only covers the 7.x range. When `@opentelemetry/sdk-node` is upgraded to 0.217.0 (the CVE fix), it pulls in `@opentelemetry/otlp-transformer@0.217.0` which depends on `protobufjs@8.0.1`. Seven GHSAs cover the 8.x range. You need a separate override: `'protobufjs@>=8.0.0 <=8.0.1': '>=8.0.2'`.

## pnpm why -r is required for exposure mapping

`pnpm audit` shows one exposure path per advisory. `pnpm why -r` shows all paths. In this release:
- opentelemetry + protobufjs CVEs reach `@shellicar/winston-azure-application-insights` via `applicationinsights` optionalDependencies. Consumer-facing.
- devalue + svelte CVEs reach `@shellicar/svelte-adapter-azure-functions` via `@sveltejs/kit` devDependencies. Dev-time only, not consumer-facing.

The other 9 packages got dependency updates only (no CVE exposure), which is why their `changes.jsonl` entries say "Updated patch dependencies" without a security section.

## Changelog script path is relative to scripts/

`pnpm --filter scripts changelog packages/<pkg>` fails because the script runs with CWD = `scripts/` and resolves the arg relative to that. Use `../packages/<pkg>` or an absolute path.
