## What npm auto-includes vs what `files` controls

npm hardcodes `README*`, `LICENSE*`, and `package.json` into every tarball regardless of the `files` field. `CHANGELOG.md` is **not** on that hardcoded list. If `files` is present and omits any `*.md` glob (or an explicit `CHANGELOG.md` entry), the changelog ships nowhere.

This is why the fix is `"*.md"` rather than `"CHANGELOG.md"`. Matches the existing convention in `core-config`, `build-clean`, etc. A single glob covers any future markdown files (e.g. `MIGRATION.md`) without further edits.

## `npm pack --dry-run` silently omits missing `files` entries

The tarball outputs from verification did **not** list `dist/` contents. That's because this worktree had not been built — there's no `dist/` on disk. npm doesn't warn about this; it just lists what actually exists. For this mission that's fine (the verification target is `CHANGELOG.md`, not the build output), but be aware: a green `pack --dry-run` doesn't prove the package is publishable, only that the glob patterns matched what was on disk at that moment.

## `svelte-adapter-azure-functions` has a `files/` subdirectory

Its `files` array entry `"files"` is a directory name, not duplication. The package ships template files (`host.json`, `local.settings.json`, `package.json`) under `files/` that consumers copy into their Azure Functions project. Easy to mistake for a typo. The append in this mission keeps it; new entry sits alongside.

## `core-di-lite` LICENSE was missing on disk

`core-di-lite` was added in 2026 (commit 93b9e36, May 2026 per recent_log). It shipped without a LICENSE file at all — npm's hardcoded LICENSE inclusion only triggers when one exists. Created with 2026 copyright year; the other packages use 2025 because they predate this year.
