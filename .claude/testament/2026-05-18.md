## What npm auto-includes vs what `files` controls

npm hardcodes `README*`, `LICENSE*`, and `package.json` into every tarball regardless of the `files` field. `CHANGELOG.md` is **not** on that hardcoded list. If `files` is present and omits any `*.md` glob, the changelog ships nowhere.

The fix is `"*.md"` rather than `"CHANGELOG.md"`. That matches the existing convention (`core-config`, `build-clean`) and covers future markdown files (e.g. `MIGRATION.md`) without further edits.

## `npm pack --dry-run` silently omits missing `dist/` entries

When the package hasn't been built, `dist/` doesn't exist on disk. npm lists only what actually exists; it doesn't warn about the gap. A green `pack --dry-run` doesn't prove the package is publishable, only that the glob patterns matched what was on disk at that moment.

## `svelte-adapter-azure-functions` has a `files/` subdirectory

Its `"files"` array entry is a directory name, not duplication. The package ships template files (`host.json`, `local.settings.json`, `package.json`) under `files/` for consumers to copy into their Azure Functions project. Easy to mistake for a typo. The `*.md` append keeps it.

## `core-di-lite` LICENSE was missing on disk

`core-di-lite` was added in 2026 (commit 93b9e36). It shipped without a LICENSE file at all — npm's hardcoded LICENSE inclusion only triggers when one exists. Created with 2026 copyright year; the other packages use 2025 because they predate this year.

## No `changes.jsonl` entries

This PR deliberately skips `changes.jsonl`. The change is packaging metadata — which files ship — not package behaviour. An entry saying "changelog is now included" would not be useful to a user reading the changelog.
