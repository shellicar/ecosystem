## Trap: `trustPolicy: no-downgrade` and `trustPolicyExclude` keying

pnpm 11's `trustPolicy: no-downgrade` fails install (`ERR_PNPM_TRUST_DOWNGRADE`) on pre-existing lockfile entries whose trust evidence dropped. Here that was chokidar@4.0.3, rollup@3.30.0, semver@5.7.2, semver@6.3.1, ua-parser-js@1.0.41. The fix is a top-level `trustPolicyExclude` (sibling of `trustPolicy`, NOT nested under `auditConfig` — the nested form silently does nothing).

Non-obvious: `trustPolicyExclude` is keyed by package NAME, so two `semver@...` entries collapse and only one version excludes. Use the multi-version form in one entry: `'semver@5.7.2 || 6.3.1'`. If a future install trips on a new package, name it the same way; if it's a second version of one already listed, fold it in with `||` rather than adding a second line.
