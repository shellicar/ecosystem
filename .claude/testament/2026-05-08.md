## Activity 3: harness migration (Stage 2 → Stage 0/1)

`.gitignore` change: `!.claude/**/*.md` → `!.claude/*/**/*.md`

The `**` pattern was matching `.claude/CLAUDE.md` directly. The `/*/**` pattern narrows it to `.md` files inside subdirectories of `.claude/` (e.g. `testament/`, `sessions/`), not `.claude/` itself. This prevents a future `git add .` from re-tracking CLAUDE.md.

`git rm --cached .claude/CLAUDE.md` was run by the SC directly (the pre-commit hook in this repo blocks `git rm` as a safety measure — correctly, per safe-operations).
