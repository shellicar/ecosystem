## Decisions

- **verbatimModuleSyntax and isolatedModules disabled**: base.json sets both to `true`, but the original tsconfig (extending `@tsconfig/node20`) had neither. The source uses bare `export { Foo }` re-exports in `src/types.ts` for types, which errors under both settings. Fixing the source is out of scope for migration; overriding in tsconfig preserves original behaviour.
- **entryNames disabled**: same pattern as build-clean. `src/types.ts` and `src/core/types.ts` would both flatten to `types` and collide.
